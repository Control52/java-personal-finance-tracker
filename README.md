Цель
Разработать консольное приложение для учета личных финансов с поддержкой нескольких счетов, операций доходов/расходов (с категориями) и переводов между счетами.
Функциональные требования
1. Управление счетами
•	Создание нового счета с указанием названия и начального баланса
•	Просмотр списка всех счетов
•	Просмотр баланса по конкретному счету
•	Удаление счета (только если баланс равен нулю)
2. Финансовые операции
•	Доход: зачисление средств на выбранный счет с указанием суммы, категории и описания
•	Расход: списание средств с выбранного счета с указанием суммы, категории и описания
•	Перевод: перемещение средств между счетами пользователя
•	Просмотр истории операций по всем счетам или по конкретному счету
3. Валидация
•	Нельзя создать операцию с отрицательной суммой
•	Нельзя выполнить расход или перевод при недостаточном балансе
•	Нельзя удалить счет с ненулевым балансом
•	Нельзя создать счет с отрицательным начальным балансом

Технические требования
Архитектура и ООП
•	Абстрактные классы: базовый класс для финансовых операций
•	Интерфейсы: для хранилища данных и сервисов
•	Исключения: кастомные проверяемые и непроверяемые исключения
•	SOLID принципы: четкое разделение ответственности
•	Maven: сборка проекта и управление зависимостями

Классы и интерфейсы
- Account (класс счета)
- Operation (абстрактный класс операции)
  - Income (класс «доход»)
  - Expense (класс «расход») 
  - Transfer (класс «перевод»)
- Storage (интерфейс хранилища)
- WalletService (интерфейс бизнес-логики)
- Custom exceptions
